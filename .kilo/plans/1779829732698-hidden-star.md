# Plan optymalizacji ciemnego motywu — Oferty PV

## Cel

Przekształcić obecny dark-mode-only CSS w spójny system zmiennych, eliminując hardcoded kolory, redukując duplikację i przygotowując bazę pod ewentualny tryb jasny w przyszłości.

## Etap 1 — Uzupełnienie zmiennych CSS w `:root`

## Etap 2 — Konwersja hardcoded kolorów w `style.css`

**Cel:** Zamienić wszystkie hardcoded kolory w `style.css` na `var(--...)`.

**Metoda:**
1. Przejść przez `style.css` (2518 linii) linia po linii.
2. Dla każdego wystąpienia koloru (hex, rgba) sprawdzić czy odpowiada którejś zmiennej.
3. Zamienić na `var(--nazwa-zmiennej)`.

**Najczęstsze zamienniki:**
| Wartość | Zmienna |
|---|---|
| `#1e293b` | `var(--border-glass)` |
| `#1e2530` | `var(--bg-tertiary)` |
| `#0a0e1a` | `var(--bg-primary)` |
| `#111827` | `var(--bg-secondary)` lub `var(--bg-card)` |
| `#94a3b8` | `var(--text-secondary)` |
| `#64748b` | `var(--text-muted)` |
| `#f1f5f9` | `var(--text-primary)` |

> ~167 zmian. Wiele to powtarzające się patterny — znacząca część to `find & replace`.

**Weryfikacja:** Strona ładuje się, kolory podstawowe nie zmieniły się wizualnie (bo wartości zmiennych są te same). Można sprawdzić diff — tylko `#hex` → `var(--)`.

## Etap 4 — Ujednolicenie zmiennych w `index.css`

**Cel:** `index.css` ma własny blok `:root` (linie 38-51) który częściowo dubluje `style.css`.

**Zmiany:**
1. Usunąć z `:root` w `index.css` zmienne które już są w `style.css`:
   - `--primary` → zastąpić `var(--accent)`
   - `--primary-hover` → zastąpić `var(--accent-hover)`
   - `--accent` (jest `#8b5cf6` czyli `--accent2`) → zastąpić `var(--accent2)`
   - `--success` → zastąpić `var(--success)`
   - `--danger` → zastąpić `var(--danger)`
   - `--warn` → zastąpić `var(--warn)`
   - `--text-dim` → zastąpić `var(--text-secondary)`
   - `--radius-sm: 10px` → zmienić użycia na `var(--radius-sm)` z `style.css` (8px) albo uznać że index.css ma własny radius — zostawić jako `--radius-sm-index: 10px`
2. Dodać do `style.css` zmienne które są unikalne w `index.css`:
   - `--glass` (jest rgba) → już dodane w Etapie 1
   - `--glass-border` → już dodane w Etapie 1
   - `--radius-lg: 24px`, `--radius-md: 16px` → dodać do `style.css`
3. Przejrzeć `index.css` pod kątem użycia swoich zmiennych i zamienić na `var(--)` z `style.css` gdzie to możliwe.

**Weryfikacja:** Strona główna (`/`) działa, kolory się nie zmieniły.

## Etap 5 — Box-shadow na zmienne

**Cel:** Zdefiniować zmienne dla cieni i zamienić hardcoded wartości.

**Zmienne do dodania w `:root`:**
```css
--shadow-sm: 0 2px 8px color-mix(in srgb, var(--shadow-color) 60%, transparent);
--shadow-md: 0 4px 16px var(--shadow-color);
--shadow-lg: 0 8px 32px var(--shadow-color);
```

## Etap 7 — Inline style w JS — część 2 (template stringi HTML)

**Cel:** Wyciągnąć inline style z template stringów HTML do osobnych klas CSS.

**Kluczowe pliki:** `offerManager.js`, `wellTransitions.js`, `wellActions.js`, `orderManager.js`

**Przykładowy wzorzec:**
```js
// PRZED:
`<button style="background:rgba(239,68,68,0.1); border:1px solid rgba(239,68,68,0.3); color:#f87171; ..."
        onmouseenter="this.style.background='rgba(239,68,68,0.25)'"
        onmouseleave="this.style.background='rgba(239,68,68,0.1)'">`

// PO:
`<button class="btn-icon btn-danger-ghost"
        onmouseenter="this.classList.add('hover')"
        onmouseleave="this.classList.remove('hover')">`
```

**Klasy CSS do dodania:**
```css
.btn-icon {
    width: 26px; height: 26px;
    border-radius: 5px;
    cursor: pointer;
    display: inline-flex;
    align-items: center;
    justify-content: center;
    transition: all 0.15s;
}
.btn-danger-ghost {
    background: var(--danger-bg);
    border: 1px solid var(--danger-border);
    color: var(--danger-hover);
}
.btn-danger-ghost.hover,
.btn-danger-ghost:hover {
    background: var(--danger-bg-hover);
}
.btn-accent-ghost {
    background: color-mix(in srgb, var(--accent) 10%, transparent);
    border: 1px solid color-mix(in srgb, var(--accent) 25%, transparent);
    color: var(--accent-text);
}
.btn-accent-ghost.hover,
.btn-accent-ghost:hover {
    background: color-mix(in srgb, var(--accent) 20%, transparent);
    border-color: color-mix(in srgb, var(--accent) 40%, transparent);
}
```

**Weryfikacja:** Interaktywne elementy (przyciski, tile) działają tak samo jak przed zmianą.

## Etap 8 — Inline style w JS — część 3 (onmouseenter/leave)

**Cel:** Wyeliminować wszystkie `onmouseenter`/`onmouseleave` z inline HTML na rzecz pseudo-klas `:hover`.

**Metoda:**
1. Dla każdego wzorca `onmouseenter="this.style.xxx='...'"` sprawdzić czy da się zastąpić klasą CSS z `:hover`.
2. Jeśli element ma dynamiczne kolory (zależne od `isActive`, `isPlaceholder` itp.), użyć `classList.toggle('active', condition)` zamiast inline handlerów.
3. W przypadkach gdzie to niemożliwe (bardzo dynamiczne wartości), dodać CSS custom properties na poziomie elementu.

**Najwięcej pracy:** `wellActions.js`, `wellTransitions.js`, `wellUI.js`, `orderManager.js`, `shared/ui.js`.

**Weryfikacja:** Hover efekty działają identycznie. Można sprawdzić poprzez audyt wizualny.

## Uwagi końcowe

- W etapach 2-4 pomocne będzie narzędzie do zamiany masowej (regex: `#XXXXXX` → odpowiednia `var(--)`)

# Plan: Tłumaczenie komentarzy angielskich na polski

## Cel

Przetłumaczyć wszystkie komentarze w języku angielskim (oraz mieszane) w kodzie źródłowym na język polski, z wyłączeniem plików testowych, konfiguracyjnych i generowanych.

## Zakres

- **17 plików** źródłowych zawiera angielskie komentarze
- **~55-60 komentarzy** do przetłumaczenia
- Języki: TypeScript, JavaScript, Python

## Zasady tłumaczenia

1. **Zachowaj techniczne terminy angielskie**, które są standardem w polskim żargonie IT:
   - `fallback` → pozostaje "fallback" lub "awaryjny"
   - `iframe`, `cache`, `token`, `snapshot` → pozostają w angielskiej formie
   - `solver`, `config`, `helper` → pozostają w angielskiej formie
   - `match`, `render`, `toggle`, `fetch` → pozostają w angielskiej formie lub tłumaczone w zależności od kontekstu

2. **Tłumacz opisową treść** na polski (całe zdania, frazy)

3. **JSDoc/duże bloki komentarzy** — tłumacz opisy, ale zostaw tagi (`@param`, `@returns` etc.) bez zmian

4. **Routing/endpointy** (`GET /api/...`) — pozostaw bez zmian (to nie są komentarze opisowe, a oznaczenia tras)

5. **Zakomentowany kod** (`// if (templateCache.has(path))`) — pozostaw bez zmian (to kod, nie komentarz)

6. **Napraw uszkodzone znaki diakrytyczne** (`�`, ``, ``, `¹`, ``, ``, ``, `` → poprawne `ąćęłńóśźżĄĆĘŁŃÓŚŹŻ`)

## Weryfikacja

Po każdej fazie:
- `npm run lint` (jeśli dostępne) — sprawdzenie składni
- `npm run typecheck` — sprawdzenie typów (komentarze nie powinny wpływać)
- Ręczne sprawdzenie diff'a dla każdego pliku

## Ryzyka

1. **Uszkodzone kodowanie** — jeśli pliki nie są w UTF-8, polskie znaki mogą się nie zapisać poprawnie. Należy sprawdzić `encoding` każdego pliku przed zapisem.
2. **Komentarze inline** — komentarze w środku linii kodu (nie `^\s*//` tylko `code // comment`) mogą zostać pominięte przez skrypty. Należy je sprawdzić ręcznie.
3. **JSDoc wieloliniowy** — bloki `/** */` mogą zawierać mieszankę angielskich i polskich fragmentów. Należy tłumaczyć tylko fragmenty opisowe, zostawiając tagi.
4. **Pliki binarne/schematy** — pomijamy `.prisma`, `.json`, `.yml`, `.html` (to nie są source code comments w ścisłym sensie).

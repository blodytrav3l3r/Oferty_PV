# Plan: Przeniesienie paska podsumowania (cena netto, koszt transportu, razem netto, % rabat) na dół strony nad dolny pasek nawigacji

## Cel

Przenieść pasek podsumowania oferty (4 kafelki: "Cena Netto", "Koszt transportu", "RAZEM Netto", "% Edytuj rabaty") z wnętrza `.rury-card` w kroku 3 na dół strony, tuż nad fixed dolny pasek nawigacji.

## Obecna struktura DOM (do zmiany)

```
wizard-step-3 (display: none/block)
  └── rury-card
        ├── rury-card-title "Pozycje oferty"
        ├── offer-items-content (tabela + transport)
        └── rury-sticky-footer ← SUMMARY BAR JEST TUTAJ
              └── rury-summary-grid (4 kafelki)
```

## Docelowa struktura DOM

```
wizard-step-3 (display: none/block)
  └── rury-card
        ├── rury-card-title "Pozycje oferty"
        └── offer-items-content (tabela + transport)
        
  rury-summary-bar (position: fixed, pokazywany tylko w kroku 3) ← PRZENIESIONY TUTAJ
    └── rury-summary-grid (te same 4 kafelki, bez zmian)

  wizard-bottom-nav (position: fixed) ← BEZ ZMIAN
```

## Uzasadnienie wyboru `position: fixed` z JS show/hide

| Kryterium | `position: sticky` | `position: fixed` |
|---|---|---|
| Zawsze na dole strony (krótka treść) | ❌ – zostaje w flow, nie dociska do dołu | ✅ – fixed zawsze na `bottom: 65px` |
| Widoczny tylko w kroku 3 (automatycznie) | ✅ – wewnątrz `wizard-step-3` | ❌ – wymaga JS show/hide |
| Treść nie chowa się za paskiem | ❌ – treść może być za fixed elementem | ❌ – też wymaga padding-bottom |

Decyzja: `position: fixed` + JS show/hide – przewaga przewidywalnego pozycjonowania.

## Modyfikacje

### 1. `public/rury.html` — zmiany HTML

**a) Usunąć** z wnętrza `.rury-card` blok `.rury-sticky-footer` (obecne linie 285–305):
```html
<!-- Usuwamy cały ten blok -->
<div class="rury-sticky-footer" style="margin: 0 -1.5rem 0; position: static; box-shadow: none; border-top: 1px solid var(--border-glass);">
    <div class="rury-summary-grid">
        <div class="rury-summary-card">...</div>
        <div class="rury-summary-card has-borders">...</div>
        <div class="rury-summary-card">...</div>
        <div class="rury-summary-card" style="border-left: 1px solid var(--border-glass); align-items: center">...</div>
    </div>
</div>
```

Po usunięciu `.rury-card` zamyka się zaraz po `offer-items-content`, a `wizard-step-3` zamyka się po `rury-card`.

**b) Dodać** między zamknięciem `wizard-step-3` a paskiem nawigacji:
```html
<div class="rury-summary-bar" id="rury-summary-bar" style="display: none;">
    <div class="rury-summary-grid">
        <!-- Te same 4 kafelki co w oryginale -->
    </div>
</div>
```

Widoczność startowa: `display: none` (ukryty do momentu aktywacji kroku 3).

### 2. `public/css/rury.css` — nowe reguły CSS

Dodać po istniejącej regule `.wizard-step.active` (linia 592):

```css
/* ── Pasek podsumowania oferty (fixed nad dolną nawigacją) ── */
.rury-summary-bar {
    position: fixed;
    bottom: 65px;
    left: 0;
    right: 0;
    z-index: 99;
    background: var(--bg-card, #111827);
    border-top: 1px solid var(--border-glass);
    border-bottom: 1px solid var(--border-glass);
    padding: 0.8rem 1.5rem;
    box-shadow: 0 -8px 30px #080808;
}

/* Większy padding-bottom dla kroku 3, by treść nie chowała się za fixed paskiem + nav */
.wizard-step#wizard-step-3.active {
    padding-bottom: 140px;
}
```

Dodać w sekcji `@media (max-width: 768px)` (około linii 605):
```css
.rury-summary-bar {
    padding: 0.6rem 0.8rem;
}
```

### 3. `public/js/rury/wizard.js` — pokazywanie/ukrywanie paska

W funkcji `goToPhase()`, po linii `updateWizardIndicator();` (linia 21), dodać:

```javascript
// Pokaż/ukryj fixed pasek podsumowania tylko w kroku 3
const summaryBar = document.getElementById('rury-summary-bar');
if (summaryBar) {
    summaryBar.style.display = step === 3 ? 'block' : 'none';
}
```

### 4. UI kafelków – BEZ ZMIAN

Wszystkie klasy używane wewnątrz summary grid pozostają takie same:
- `.rury-summary-grid` – grid 4-kolumnowy
- `.rury-summary-card` – kafelek
- `.rury-summary-card.has-borders` – kafelek z ramkami bocznymi
- `.rury-summary-label` – etykieta
- `.rury-summary-value`, `.rury-summary-value.accent`, `.rury-summary-value.warn` – wartości
- `.rury-summary-value-lg` – duża wartość (RAZEM Netto)
- `.rury-summary-sub` – podpis (Brutto, szczegóły transportu)
- `.rury-btn-discount` – przycisk "% Edytuj rabaty"

### 5. JavaScript aktualizujący wartości – BEZ ZMIAN

`updateOfferSummary()` w `transport.js` (linie 63-107) aktualizuje `sum-netto-products`, `sum-transport-cost`, `sum-total-netto` przez `getElementById` – działa niezależnie od pozycji DOM elementów.

`showItemDiscountModal()` w `offerExports.js` (linia 193) – wywoływany przez `onclick` na przycisku, działa niezależnie od pozycji.

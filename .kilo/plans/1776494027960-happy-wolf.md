# Plan: Offline Installation Package for WITROS Oferty

## Cel - Utworzenie Pakietu Offlineowego do Instalacji

Aby umożliwić instalację aplikacji **WITROS Oferty** bez połączenia z internetem, należy przygotować kompletny pakiet z wszystkimi wymaganymi aplikacjami i programami.

## **1. Wymagania Podstawowe**

### Programy wymagane do zainstalowania:

- **Node.js** (wersja 18 lub nowsza) - środowisko uruchomieniowe JavaScript
- **npm** (zawarty w Node.js) - menedżer pakietów
- **Python 3.x** - tylko jeśli potrzebne są funkcje AI (backend optymalizacji)
- **Docker** (opcjonalnie) - do wdrożenia w kontenerze

### Narzędzia do budowania:

- Kompilator TypeScript
- Moduły budowania projektu (zdefiniowane w package.json)

## **2. Struktura Pakietu Offlineowego**

Katalog `offline_install_package/` powinien zawierać:

```
offline_install_package/
├── nodejs/                    # Node.js installer (wersja 18+)
├── python/                    # Python installer (jeśli potrzebne)
├── docker/                    # Docker Desktop installer
├── project_files/             # Pełny kod projektu:
│   ├── package.json
│   ├── server.ts
│   ├── prisma/
│   ├── public/
│   └── ...
├── npm_packages/              # Pobrane pakiety npm (cache)
├── requirements.txt           # Wymagania Python (jeśli potrzebne)
├── README_offline_install.md  # Instrukcja instalacji offline
└── install.bat / install.sh   # Skrypt instalujący
```

## **3. Instrukcja Pakietowania**

### Krok 1: Pobierz wszystkie zależności

```bash
# Na komputerze z dostępem do internetu:
cd project_directory
npm install --cache ../npm_packages --prefer-offline

# Zapisz cache pakietów npm
```

### Krok 2: Przygotuj kompletny kod projektu

- Skopiuj wszystkie pliki projektu
- Uwzglębnij pliki: package.json, server.ts, prisma/, public/, etc.
- Dodaj plik .env.example jako przykład konfiguracji

### Krok 3: Utwórz skrypty instalacyjne

- **install.bat** (Windows):

    ```batch
    @echo off
    echo Instalacja WITROS Oferty w trybie offline...
    echo Upewnij sie, ze Node.js jest zainstalowane
    cd %~dp0
    npm ci --offline --cache ..\npm_packages
    npx prisma generate
    npm run build
    echo Instalacja zakonczona!
    ```

- **install.sh** (Linux/macOS):
    ```bash
    #!/bin/bash
    echo "Instalacja WITROS Oferty w trybie offline..."
    echo "Upewnij sie, ze Node.js jest zainstalowane"
    cd "$(dirname "$0")"
    npm ci --offline --cache ../npm_packages
    npx prisma generate
    npm run build
    echo "Instalacja zakonczona!"
    ```

## **4. Opcjonalne: Python AI Backend**

Jeśli wymagane są funkcje AI:

- Dodaj folder `well_configurator_backend/` z aplikacją Python
- Dołącz requirements.txt z zależnościami
- Użytkownik może uruchomić: `python well_configurator_backend/run.py`
- Alternatywnie: uruchom `start_all.bat` (łączy Node.js + Python)

## **5. Bezpośrednia Instalacja Offline**

### **Windows:**

1. Uruchom `install.bat` jako administrator
2. Serwer będzie dostępny pod adresem: http://localhost:3000

### **Linux/macOS:**

1. Uruchom: `chmod +x install.sh`
2. Uruchom: `./install.sh`
3. Serwer będzie dostępny pod adresem: http://localhost:3000

## **6. Weryfikacja Instalacji**

Po instalacji sprawdź:

- ✅ Node.js: `node --version`
- ✅ npm: `npm --version`
- ✅ Baza danych: `data/app_database.sqlite`
- ✅ Skompilowane pliki: `dist/`
- ✅ Dostępność interfejsu: http://localhost:3000

## **7. Zalecenia**

- **Przed instalacją:** Upewnij się, że wszystkie zależności są w cache npm
- **Kompresja:** Można spakować katalog `offline_install_package/` w archiwum ZIP
- **Aktualizacja:** Do aktualizacji potrzebny jest dostęp do internetu lub nowy pakiet
- **Dokumentacja:** Dołącz plik README_offline_install.md z szczegółowymi instrukcjami

## **Zalety Pakietu Offlineowego:**

- 🚀 Szybsza instalacja (brak pobierania z npm)
- 🔒 Prywatność (dane nie wychodzą z sieci)
- 📦 Jednorazowa konfiguracja
- 🔄 Łatwa aktualizacja pakietu
